# AUDIT_WP-2026-133 - Skill references scaffold for validator

## Audit Focus

Verify that the 7 skills currently failing `skills/validate_all.py` are repaired by adding a minimal `references/.gitkeep` placeholder, without changing the validator logic itself.

## What Must Be True

- The fix is structural only.
- The validator still checks for `references/` and no code path is changed.
- Each affected skill contains a `references/` directory.
- Each affected skill preserves that directory in Git via `.gitkeep`.
- No additional skill content is introduced as part of the fix.
- The ticket does not mutate the validator contract or introduce new skill semantics.
- `deliverable_type` is `documentation`, not `code`.

## Failure Modes

1. A skill still lacks `references/` after the change.
2. The fix adds README or other content instead of the minimal placeholder.
3. The validator script is modified to stop checking `references/`.
4. Only some of the failing skills are fixed, leaving partial validation failures.
5. The ticket expands into a larger skill hygiene refactor instead of the targeted structural fix.

## Review Checklist

- Confirm all 7 skill directories have `references/.gitkeep`.
- Confirm `skills/validate_all.py` is untouched.
- Confirm the validator passes after the change.
- Confirm the fix stays minimal and does not add unrelated skill content.
