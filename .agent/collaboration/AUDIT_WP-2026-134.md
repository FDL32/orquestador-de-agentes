# AUDIT_WP-2026-134 - Anchor Manager review diff to branch base and add provenance

## Verificacion inicial
El review bridge ya empaqueta contexto canonical, pero hoy su diff sale de `git diff HEAD`, que puede quedar vacio una vez el Builder commitea. El gap real es de anclaje de diff y trazabilidad, no de state machine.

## Hallazgos
- `bus/review_bridge.py` construye el diff con `git diff HEAD` y el diff-stat con el mismo anclaje al working tree.
- El prompt del Manager ya contiene un bloque de instrucciones, pero no declara de forma explicita que la review es advisory.
- No existe una seccion compacta de provenance del commit en el prompt actual.
- El ticket necesita cubrir el caso en que `origin/main` no sea alcanzable o no exista en el entorno, degradando a `git diff HEAD` con warning visible.
- Los tests del bridge tendran que validar el nuevo origen del diff, el fallback y la presencia de provenance sin cambiar `DECISION`.

## Riesgo observado
Si el ticket intenta meter schema JSON estructurado o panel multi-reviewer, el alcance se amplifica y se aleja del objetivo principal: revisar el diff real del ticket con trazabilidad.

## Verificacion esperada
- El Manager recibe un diff anclado a `origin/main...HEAD` cuando el remote base esta disponible.
- Si `origin/main` no es accesible, el prompt muestra un warning y degrada a `git diff HEAD`.
- El prompt incorpora provenance del ultimo commit del ticket.
- La instruccion advisory queda explicita.
- La salida canonica del review sigue siendo `DECISION: APPROVE / CHANGES / INSPECT`.
