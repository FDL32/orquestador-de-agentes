# PLAN_WP-2026-139 - Cached canonical anti-pattern inventory for review_bridge

## Goal

Make `bus/review_bridge.py` load the canonical anti-pattern inventory from `skills/_shared/anti-patterns.md` once, cache it in the bridge instance, and compose the Manager rubric from that cached source instead of an inline duplicate list.

## Strategy

1. Add a cached loader in `ReviewBridge.__init__()` for the canonical AP inventory.
2. Refactor `_rubric_for_type()` to render AP-01..AP-07 from the cached inventory.
3. Keep the rubric base, dynamic learnings, and review decision contract unchanged.
4. Add tests for single-load behavior, rubric composition, and safe fallback when the inventory file is unavailable.

## Scope Notes

- The canonical inventory remains in `skills/_shared/anti-patterns.md`.
- `code-rules.md` and `review-checklist.md` stay as derived views.
- This ticket does not change the observation pipeline or the closeout pipeline.
- The cache should avoid file I/O on every review prompt build.
- Any future AP-08 extends the shared inventory first, then the derived views.

## Acceptance Shape

- `ReviewBridge` reads the canonical AP inventory from file once per instance.
- `_rubric_for_type()` uses the cached inventory instead of duplicating AP text inline.
- The Manager rubric still includes AP-01..AP-07.
- The prompt remains functional if the inventory file is missing or unreadable.
- Tests cover loading, caching, composition, and the safe fallback path.
