# Audit - WP-2026-159

## Scope
- `bus/supervisor.py`: autoridad de requeue y relanzado reactivo.
- `tests/test_supervisor.py`: regresiones y smoke path del supervisor reactivo.

## Blocks
- Blocker si el ticket intenta arreglar el problema sin instrumentacion previa y no deja observable el motivo real del relanzado omitido o fallido.
- Blocker si `run_once()`/`run_reactive()` no relanza Builder automaticamente cuando aparece un nuevo antecedente de `CHANGES`.
- Blocker si dos eventos `CHANGES` consecutivos pueden duplicar relanzado o inflar `loop_current_round`.
- Blocker si `READY_TO_CLOSE`, `COMPLETED` o `HUMAN_GATE` permiten relanzado.
- Blocker si los tests del ciclo reactivo usan subprocess real o terminales reales en lugar de `tmp_path` y mocks controlados.
- Blocker si no se persiste ninguna traza legible del intento de relanzado (estado, resultado o motivo de skip/fallo).
- Blocker si el plan no fija el payload de `BUILDER_RELAUNCH_ATTEMPTED` o deja el seam del launcher a criterio del Builder.
