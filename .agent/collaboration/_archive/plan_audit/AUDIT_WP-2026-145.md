# AUDIT_WP-2026-145 - Deterministic STATE projection probe

## Audit Focus

Verify that a read-only probe can reconstruct the active ticket state from `events.jsonl` and compare it against `STATE.md` without mutating canonical collaboration files.

## What Must Be True

- The probe derives state from the event bus, not from handwritten markdown.
- The probe reports drift when the bus and `STATE.md` disagree.
- `TURN.md` remains under controller ownership and is not rewritten by the probe.
- The implementation reuses `bus/state_machine.py` for transition semantics.
- The experiment remains read-only and isolated to the probe plus tests.

## Failure Modes

1. The probe rewrites canonical files instead of reporting only.
2. The probe reimplements transition logic and diverges from the bus state machine.
3. The probe cannot handle missing or empty event streams.
4. Drift exists but is not reported clearly.
5. The ticket scope expands into controller integration before the experiment is validated.

## Review Checklist

- Confirm the probe is read-only.
- Confirm the state machine remains the source of transition truth.
- Confirm drift reporting is explicit and deterministic.
- Confirm tests cover matching, drift and empty/missing bus cases.
