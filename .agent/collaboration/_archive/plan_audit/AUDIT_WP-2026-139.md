# AUDIT_WP-2026-139 - Cached canonical anti-pattern inventory for review_bridge

## Audit Focus

Verify that `review_bridge.py` loads the canonical anti-pattern inventory from `skills/_shared/anti-patterns.md` once, caches the ID/name index, and uses that cache to compose the Manager rubric while keeping the detailed detection guidance in code.

## What Must Be True

- AP-01..AP-07 come from the shared canonical inventory as IDs + names.
- The bridge does not re-read the inventory file for every prompt build.
- The rubric composition still preserves the base review instructions, the AP detail blocks, and the dynamic learnings block.
- The review prompt remains valid if the shared inventory file is unavailable.
- The change does not alter the `APPROVE / CHANGES / INSPECT` contract.
- The derived views remain derived, not authoritative.
- The bridge resolves the inventory path relative to `bus/review_bridge.py`, not via `project_root`.
- Missing inventory files emit a warning and omit the AP inventory section.

## Failure Modes

1. The bridge still embeds the AP list inline and the shared file becomes decorative.
2. The inventory file is read on every review prompt build.
3. A missing inventory file crashes review transport.
4. The cached inventory is stale and drifts from `skills/_shared/anti-patterns.md`.
5. The rubric loses the dynamic learnings block or the review decision contract.

## Review Checklist

- Confirm the AP inventory is loaded once and cached on bridge initialization.
- Confirm `_rubric_for_type()` consumes the cached inventory section.
- Confirm the Manager prompt still includes AP-01..AP-07.
- Confirm the fallback path keeps review transport alive.
- Confirm tests cover caching, composition, and missing-file safety.
