# PLAN WP-2026-125 - Launcher multi-root

## Contexto
WP-C completa el desacople motor/destino. El launcher debe resolver el workspace destino antes de abrir Supervisor, Bridge y Builder.

## Estrategia
Opcion seleccionada: launcher multi-root con contrato de precedencia:
`--project-root` > `AGENT_PROJECT_ROOT` > `motor_destination_link.json` > fallback local.
Si no hay CLI ni env, el launcher lee `.agent/config/motor_destination_link.json` del workspace actual y toma `destination_root`.

## Alcance
- `scripts/launch_agent_terminals.ps1` resuelve y propaga el root destino.
- `tests/test_launch_agent_terminals_script.py` cubre la precedencia y el arranque.
- Sin import circular.
- `WP-mini` de higiene sigue fuera del alcance.

## Fases
1. Auditar la resolucion del launcher y el origen del root destino.
2. Implementar la propagacion del root resuelto a los procesos hijos.
3. Validar el arranque multi-root y registrar el estado canonico.

## Entregable
Launcher multi-root operativo y documentado, listo para arrancar contra un workspace destino externo.

## Riesgos
- Romper el arranque local si la precedencia no se aplica de forma estricta.
- Introducir ambiguedad entre motor y destino si el launcher no resuelve el root antes de lanzar procesos.
