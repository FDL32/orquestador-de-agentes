# PLAN_WP-2026-141 - Google eng-practices review standards alignment

## Goal

Adapt a small subset of `google/eng-practices` into the repo's documentation flow so review decisions become more objective, clearer, and easier to audit.

## External Sources

- https://google.github.io/eng-practices/review/reviewer/standard.html
- https://google.github.io/eng-practices/review/developer/small-cls.html
- https://google.github.io/eng-practices/review/reviewer/comments.html

## Strategy

1. Add a positive approval principle to the review checklist: do not block a change when it improves code health, even if it is not perfect.
2. Add a `Nit` convention for non-blocking comments so suggestions stay distinct from blockers.
3. Add a closure-criteria traceability line to `AGENTS.md` that points back to the approval principle.
4. Add a `CREDITS.md` attribution row for `google/eng-practices` with the correct license and adaptation note.
5. Keep the scope documentation-only and avoid any production code changes.

## Scope Notes

- This ticket is intentionally narrow and does not modify runtime behavior.
- The goal is to improve review quality and shared vocabulary, not to create a new process surface.
- The existing review checklist remains the primary place where Manager-facing review criteria live.
- The external references should be linked directly, not mentioned only by name.

## Acceptance Shape

- `review-checklist.md` contains the approval principle as a positive rule.
- `review-checklist.md` or the bridge prompt clearly defines `Nit` as non-blocking.
- `AGENTS.md` includes a traceability line that references the approval principle.
- `CREDITS.md` includes a row for `google/eng-practices` with CC-BY 3.0 attribution.
- The ticket remains documentation-only.
