# AUDIT_WP-2026-136 - Semantic --candidates input for session_close_observations

## Verificacion inicial

`session_close_observations.py` ya tiene un pipeline completo de validacion,
filtrado y append (`validate_schema`, `is_duplicate`, `process_candidates`,
`append_observations`). Solo acepta candidatos auto-generados via `--ticket`,
que extrae dos observaciones fijas del `work_plan.md` activo. No hay forma de
inyectar candidatos externos. `--ticket` esta marcado como `required=True` en
argparse.

## Hallazgos

- El modo `--ticket` genera candidatos demasiado genericos: siempre el mismo
  par de senales (ticket-completion + architecture decision).
- No existe un punto de entrada para candidatos semanticamente ricos.
- `load_candidates_from_file` no existe.
- El parser actual (`required=True` en `--ticket`) impide anadir `--candidates`
  sin refactorizar el grupo de argumentos. La forma correcta es
  `add_mutually_exclusive_group(required=True)`, no validacion manual.
- `load_existing_observations()` abre `observations.jsonl` con
  `encoding="utf-8"` sin `errors="replace"`, igual que el bug ya corregido en
  `pre_compact_hook.py` (WP-2026-135). Un archivo con bytes invalidos lanza
  `UnicodeDecodeError` y rompe el cierre de sesion.
- `skills/session-close-observations/SKILL.md` solo documenta `--ticket`.

## Riesgos a acotar

- No corromper el pipeline existente: `process_candidates` y `append_observations`
  no deben tocarse.
- `load_candidates_from_file` no debe re-validar schema: eso ya lo hace
  `validate_schema`. Solo valida estructura de carga (lista de dicts).
- Elementos no-dict en el JSON de entrada deben saltarse con warning, no abortar
  el lote entero.
- `source_ticket` ausente es un error de schema, no de carga.
- El endurecimiento de `load_existing_observations` sigue el mismo patron que
  `pre_compact_hook.py`: `errors="replace"` en el `open()`, sin mas cambios.

## Verificacion esperada

- `add_mutually_exclusive_group(required=True)` impone la exclusion a nivel de
  argparse, sin logica manual en `main()`.
- `load_candidates_from_file` lanza excepciones tipadas para: archivo ausente,
  UTF-8 invalido, JSON roto, top-level no-lista.
- `load_existing_observations` tolera `observations.jsonl` con bytes invalidos.
- `main()` despacha a la rama correcta segun el flag activo.
- 6 tests nuevos cubren todos los casos relevantes.
- `SKILL.md` refleja el nuevo modo y la exclusion mutua.
- `ruff` y `pytest` pasan en verde.
