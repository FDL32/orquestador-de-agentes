# Audit - WP-2026-160

## Lo que ya existe
- `AP-13` ya documenta el patron de proceso de larga vida que sigue ejecutando codigo viejo tras un hot-patch.
- `scripts/launch_agent_terminals.ps1` ya tiene el camino `-ResumeBuilder`, la limpieza de locks y el arranque de ventanas.
- `tests/test_launch_agent_terminals_script.py` ya valida el contenido del launcher y puede servir como base de regresion.
- `bus/supervisor.py` ya tiene requeue, watermark y relanzado observable; el hueco es la salida cooperativa tras el handoff para no dejar un supervisor viejo con autoridad viva.

## Blockers
- `-ResumeBuilder` no garantiza que el supervisor en memoria quede fresco antes de abrir Builder -> el ticket debe introducir una rama explicita de reinicio/renovacion del supervisor en el launcher y una salida cooperativa del supervisor viejo tras el handoff.
- El camino de requeue puede seguir confiando en un supervisor viejo si `run_reactive()` no sale tras el requeue y el launcher no espera a que el lock desaparezca -> el ticket debe implementar la secuencia completa: salida cooperativa del supervisor viejo, poll del launcher hasta lock liberado, arranque del supervisor nuevo, apertura de Builder.
- No hay trazabilidad observable del reinicio del supervisor -> el ticket debe dejar stderr explicito en el launcher y un evento `SUPERVISOR_RESTARTED` con payload `{"round": N, "reason": "resume-builder"}` emitido por el supervisor nuevo al arrancar; no hay evento ad hoc desde el launcher.
- Si el fix se implementa con un camino ad hoc distinto al arranque normal del launcher, el alcance se ha desviado -> el ticket debe reutilizar el mismo patron de arranque que ya usa el launcher para el supervisor.

## Evidencia esperada
- Test de contenido del launcher: `-ResumeBuilder` llama al helper de espera antes de abrir Builder.
- Test de contenido del launcher: el camino falla cerrado con stderr explicito y exit code no cero si el poll vence sin que el lock desaparezca.
- Test de unidad del supervisor: `run_reactive()` devuelve tras el primer requeue exitoso sin continuar el bucle.
- Test de contenido del supervisor: el arranque fresco emite `SUPERVISOR_RESTARTED` con `{"round": N, "reason": "resume-builder"}`.
- `pytest`, `pytest-safe` y `agent_controller --validate` verdes.
