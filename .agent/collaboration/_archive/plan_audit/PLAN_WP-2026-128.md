# PLAN WP-2026-128 - Skill allowlist contract hardening

## Contexto
WP-2026-128 endurece el contrato de skills por rol para que el Manager, Builder y Supervisor solo vean skills permitidas por configuracion valida y trazable.

## Estrategia
Primero externalizar y versionar las allowlists por rol en `agents.json` usando la key `skill_allowlists`. Despues validar que las allowlists solo referencian skills existentes descubiertas por `discover_skills.py` y que el resolver aplica permisos de forma consistente, fallando si el catalogo queda vacio. Por ultimo verificar que el prompt del Manager y los tests consumen solo el catalogo filtrado.

## Alcance
- `.agent/agents_config.py` valida la configuracion y versiona allowlists por rol.
- `.agent/config/agents.json` declara allowlists opcionales para roles.
- `bus/skill_resolver.py` resuelve, filtra y valida skills por rol y debe manejar el caso de catalogo vacio de forma explicita.
- `bus/review_bridge.py` consume solo el catalogo filtrado en el prompt del Manager.
- `bus/exceptions.py` expone errores claros para allowlists o skills invalidos.
- `tests/unit/test_agents_config.py`, `tests/unit/test_skill_discovery.py` y `tests/test_manager_review_bridge.py` cubren configuracion, resolucion y prompt.

## Fases
1. Externalizar las allowlists por rol en `agents.json` con la key `skill_allowlists` y compatibilidad hacia atras.
2. Validar allowlists y referencias de skills de forma temprana, cruzandolas contra el catalogo real de `discover_skills.py` y tratando el catalogo vacio como error de infraestructura.
3. Verificar que el prompt del Manager y las quality gates reflejan solo el catalogo filtrado.

## Entregable
Un contrato de skills por rol configurado, validado y reflejado en el prompt del Manager sin exponer skills no permitidas.

## Riesgos
- Dejar allowlists huérfanas que rompan la carga de configuracion.
- Introducir incompatibilidad con configuraciones legacy sin allowlists declaradas.
- Dejar que discover_skills falle en silencio y convierta el contrato en falsos positivos de skills inexistentes.
- Exponer demasiada logica de permisos en el prompt en vez de mantenerla en el resolver.
