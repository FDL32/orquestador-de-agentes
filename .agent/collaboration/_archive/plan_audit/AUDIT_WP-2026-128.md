# AUDIT WP-2026-128 - Skill allowlist contract hardening

## Objetivo de auditoria
Verificar que las skills por rol estan filtradas por configuracion valida, con allowlists trazables y prompt del Manager alineado al catalogo permitido.

## Criterios funcionales
- `agents.json` puede declarar allowlists por rol usando la key `skill_allowlists` sin romper compatibilidad hacia atras.
- `SkillResolver` filtra skills por rol y valida accesos de forma consistente.
- Las referencias a skills inexistentes quedan detectadas por validacion o tests.
- Un catalogo de skills vacio se reporta como error de infraestructura, no como allowlist huérfana.
- El prompt del Manager solo inyecta el catalogo filtrado para su rol.

## Criterios arquitectonicos
- La logica de permisos vive en el resolver, no en el bridge de review.
- La configuracion queda versionada y validada desde `agents_config.py`.
- Los errores de allowlist invalida son explicitos y auditables.
- `create_resolver()` o la capa de validacion deben evitar que un `ImportError` silencioso deje el catalogo vacio sin aviso.

- El prompt del Manager y el resolver comparten un unico contrato de skills permitidas.
