# Audit - WP-2026-173

## Lo que ya existe
- `scripts/pre_handoff_guard.py` ya puede detectar checkpoint M3 ausente o desalineado.
- `agent_controller.py` ya tiene la secuencia de `--mark-ready` y el guard previo.
- `tests/test_agent_controller.py` existe y cubre funciones canonicas del controlador.

## Riesgos
- Si el helper no lee `Files Likely Touched`, puede stagear demasiado o demasiado poco.
- Si el helper no crea/renueva el checkpoint inline, el problema de 171/172 reaparece.
- Si los tests no cubren el caso de arbol sucio filtrando superficies vivas, ausencia de cambios e idempotencia, el helper puede quedarse ambiguo.
- Si los hooks de `git commit` fallan y el stderr no se propaga, el operador no vera la causa real del fallo.

## Evidencia esperada
- Un test del camino feliz `commit -> checkpoint -> clean`.
- Un test que pase sin cambios cuando el tag ya esta alineado.
- Un test que falle cuando no hay cambios de entrega que preparar pero el tag necesita refresh.
- Un test que falle cuando el arbol no queda limpio al final despues de filtrar superficies vivas.
- Un test que verifique que el stderr de un hook fallido se muestra tal cual.
