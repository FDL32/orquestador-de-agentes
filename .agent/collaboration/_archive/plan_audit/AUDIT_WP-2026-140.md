# AUDIT_WP-2026-140 - Bus import boundary test for scripts dependency firewall

## Audit Focus

Verify that the new boundary test statically inspects `bus/` and prevents new dependencies on `scripts/` beyond the approved seam `scripts.discover_skills`.

## What Must Be True

- The boundary test does not rely on runtime imports that could hide side effects.
- The allowlist is explicit and minimal.
- The test reports the offending `bus` module and `scripts.*` import when the boundary is violated.
- The current tree remains valid with the approved seam.
- The change does not alter production behavior.

## Failure Modes

1. The test is too broad and flags the existing `scripts.discover_skills` seam as a failure.
2. The test is too narrow and misses new `scripts.*` imports in `bus/`.
3. The test imports modules with side effects and becomes flaky.
4. The failure message is vague and does not identify the exact offending module or import.

## Review Checklist

- Confirm the test enumerates `bus/` imports statically.
- Confirm `scripts.discover_skills` is the only allowed import from `scripts/`.
- Confirm the failure message names the exact offending module and import.
- Confirm the test is deterministic and has no side-effectful imports.
