# PLAN_WP-2026-140 - Bus import boundary test for scripts dependency firewall

## Goal

Add a narrow regression test that enforces the `bus/` layer only imports the permitted seam `scripts.discover_skills` and rejects any other `scripts.*` dependency.

## Strategy

1. Scan the `bus/` tree statically for import statements that target `scripts.*`.
2. Allow only `scripts.discover_skills` as the approved seam.
3. Report the exact `bus` module and forbidden import when the boundary is violated.
4. Keep the test isolated, deterministic, and free from runtime side effects.

## Scope Notes

- This ticket targets architecture drift, not production code behavior.
- The allowed seam exists because `bus/skill_resolver.py` already relies on `scripts.discover_skills`.
- The test should guard against new imports from `bus/` into `scripts/` without forcing a redesign of the existing seam.
- No production modules should change unless the boundary test exposes a previously unrecognized exception.
- The test must work with AST/static inspection rather than importing all `bus` modules.

## Acceptance Shape

- `tests/test_bus_boundary.py` exists and scans `bus/` imports statically.
- The test passes with the current allowlist.
- Any new `scripts.*` import added under `bus/` fails the test with a clear message.
- The validation gates pass for the new boundary test.
