# AUDIT_WP-2026-144 - Destination ticket prefix onboarding

## Audit Focus

Verify that destination projects can declare and persist a local ticket namespace without affecting the motor's `WP-YYYY-NNN` convention.

## What Must Be True

- The installer can record a destination prefix when `--prefix XXX` is provided.
- The destination link metadata includes the prefix for traceability.
- `--validate` warns when a host-project lacks `Ticket prefix:` in `PROJECT.md`.
- The bootstrap prompt and repo docs use the destination namespace examples consistently.
- Existing validation and ticket lifecycle behavior remains intact.
- The link file remains a generated destination artifact; the motor only updates the writer.

## Failure Modes

1. The installer still requires a manual metadata edit after `--install`.
2. Destination projects can start tickets without declaring a local prefix.
3. The link metadata does not capture the namespace, making audits harder.
4. The bootstrap prompt or docs still suggest `WP-YYYY-NNN` for destination repos.
5. The changes disturb the current canonical validation path.
6. The Builder is pointed at a nonexistent integration test file instead of a new prefix-warning test.

## Review Checklist

- Confirm the installer accepts and persists the local prefix.
- Confirm the destination link file records the prefix.
- Confirm `--validate` warns on missing prefix for host-project destinations.
- Confirm the docs point to `XXX-YYYY-NNN` for destinations and `WP-YYYY-NNN` for the motor.
