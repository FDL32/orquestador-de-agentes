# AUDIT WP-2026-125 - Launcher multi-root

## Objetivo de auditoria
Confirmar que el launcher resuelve el workspace destino externo de forma canonica y sin romper el arranque local.

## Criterios funcionales
- La precedencia de root es: `--project-root` > `AGENT_PROJECT_ROOT` > `motor_destination_link.json` > fallback local.
- Si no hay override externo, el launcher usa `motor_destination_link.json` como fallback de destino.
- Supervisor, Bridge y Builder reciben el root resuelto antes de iniciar.
- El flujo local sigue funcionando si no hay destino externo.

## Criterios arquitectonicos
- No se introduce import circular.
- El launcher permanece como punto de resolucion.
- El enlace motor-destino se usa como contrato de configuracion, no como logica de negocio.
