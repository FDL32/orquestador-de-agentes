# PLAN_WP-2026-154 - Strictness profiles and live guard_paths activation

## Contexto
`guard_paths.py` contiene la logica de bloqueo, pero su `__main__` actual no la usa y devuelve `{"continue": True}` siempre. Al mismo tiempo, `agents.json` ya tiene `active_profile`, lo que deja espacio para una nueva capa de strictness sin romper la configuracion existente.

## Objetivo tecnico
Conectar el hook a la logica real y parametrizarlo por perfil. El cambio debe ser compatible hacia atras, introducir una migracion simple y mantener como superficie viva `execution_log.md` y los otros archivos canonicos del flujo.

## Estrategia
0. Verificar `pytest --collect-only` sobre los dos tests objetivo; si hay `ImportError`, fijar la precedencia de imports en `tests/conftest.py` como correccion aislada de tests.
1. Añadir `strictness_profile` y `profiles` en `agents.json` con `minimal`, `standard` y `strict`.
2. Migrar `schema_version` de `1.1` a `1.2` con backfill a `standard`.
3. Mantener `guard_paths.py` self-contained: leer `agents.json` directamente, emitir razones a `stderr`, y responder con `exit(0)` / `exit(2)` segun allow/block.
4. Cubrir el comportamiento con tests de migracion, perfiles, salida del hook y el contrato strict.

## Diseño de perfiles
- `minimal`: solo la superficie sensible minima.
- `standard`: replica la proteccion actual como default.
- `strict`: endurece el guard y el scope gate con una blacklist de comandos destructivos mas amplia y bloqueo de escrituras fuera de `Files Likely Touched`, pero no convierte `execution_log.md` en una ruta globalmente bloqueada.

## Detalles de implementacion
- Unknown profiles should fail closed with a clear reason.
- The hook should reuse the existing helper functions instead of duplicating matching logic.
- Any profile-specific extra patterns should come from config, not hardcoded branch sprawl.
- Security logging should remain best-effort and non-fatal.
- `agents_config.py` should keep its import path robust in tests, even if `.agent/runtime` shadows the root `runtime` package; the prescribed fix is a `tests/conftest.py` that prepends the project root to `sys.path` — do not modify production code for this.

## Riesgos
- If the profile schema is too strict, the hook could lock out normal work.
- If the hook keeps fallbacking to `continue`, the change is cosmetic and unsafe.
- If the migration does not backfill existing configs, older workspaces will fail validation.
- If the runtime import precedence is wrong in pytest, the config loader will fail before the new profile logic is exercised.
