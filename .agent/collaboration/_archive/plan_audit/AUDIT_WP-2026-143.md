# AUDIT_WP-2026-143 - Bus-backed mark-ready idempotency

## Audit Focus

Verify that `--mark-ready` uses the bus as the authority for idempotency and does not re-open a review cycle when the ticket is already ready or already terminal.

## What Must Be True

- If the bus says `READY_FOR_REVIEW`, `--mark-ready` behaves like the current already-ready no-op.
- If the bus says `READY_TO_CLOSE` or `COMPLETED`, `--mark-ready` returns cleanly without emitting duplicate `BUILDER_EXIT` or `STATE_CHANGED` events.
- If the bus is unavailable, the existing markdown fallback path still works.
- Existing emission-order tests stay green.
- The new tests cover both the bus-backed no-op and the fallback path.

## Failure Modes

1. `--mark-ready` still relies on markdown state and repeats the review cycle after drift.
2. READY_TO_CLOSE or COMPLETED still cause duplicate Builder emissions.
3. The bus-backed guard breaks the existing READY_FOR_REVIEW idempotent path.
4. The fallback path regresses when the bus is unavailable.
5. Existing bus emission tests fail because the guard changes the normal closeout path.

## Review Checklist

- Confirm the guard reads bus state before the markdown branch.
- Confirm terminal-adjacent states are no-op outcomes.
- Confirm the fallback path remains intact.
- Confirm tests cover READY_FOR_REVIEW, READY_TO_CLOSE, COMPLETED and bus-unavailable cases.
