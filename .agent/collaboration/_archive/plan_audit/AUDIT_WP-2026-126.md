# AUDIT WP-2026-126 - Bus end-to-end review validation

## Objetivo de auditoria
Confirmar que el transporte de `review packet` lleva un prompt real al Manager y que el bus/proyecciones llegan a un estado final coherente.

## Criterios funcionales
- `opencode run` recibe un mensaje posicional corto.
- El contexto pesado viaja en un paquete de review por archivo.
- El Manager devuelve `approve`, `changes` o `inspect` reales, no la ayuda del CLI.
- El bus y las proyecciones reflejan la misma decision final.

## Criterios arquitectonicos
- No se usa `--file` como prompt.
- El paquete de review queda asociado al ticket y al intento.
- El transporte sigue siendo compatible con Windows.

## Verificacion esperada
- `pytest`
- `ruff check .`
- `python .agent/agent_controller.py --validate --json --force`

## Archivos esperados
- `bus/review_bridge.py`
- `scripts/manager_review_bridge.py`
- `tests/test_review_bridge.py`
- `tests/test_manager_review_bridge.py`
- `tests/test_review_cycle_e2e.py`

## Resultado esperado
- Review real del Manager.
- Bus y proyecciones sincronizados.
- Cierre canonico o bloqueo con evidencia.
