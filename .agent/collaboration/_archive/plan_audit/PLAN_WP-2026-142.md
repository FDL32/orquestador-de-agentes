# PLAN_WP-2026-142 - Symmetric mark-ready scope gate

## Goal

Harden `--mark-ready` so the Builder cannot pass the scope gate by declaring touched files that never appeared in the diff. The change preserves the current out-of-scope block and adds symmetric coverage checks.

## Strategy

1. Extend `check_scope_gate` to evaluate both `changed_files - whitelist` and `whitelist - changed_files`.
2. Keep the current out-of-scope behavior unchanged.
3. Block closeout when none of the declared files appear in the diff.
4. Warn, but do not block, when only some declared files appear in the diff.
5. Keep the no-git and empty-whitelist cases unchanged.
6. Thread the richer result into `--mark-ready` and verify the end-to-end behavior with tests.

## Scope Notes

- The `events.jsonl` exclusion remains untouched.
- Circuit-breaker logic remains untouched.
- The Manager semantic-injection path is out of scope for this ticket.

## Acceptance Shape

- `check_scope_gate` returns enough signal to distinguish block vs warning.
- Zero-overlap declared files fail closeout.
- Partial overlap warns and continues.
- Existing current-path tests still pass.
- `--mark-ready` remains consistent with the bus emission tests.
