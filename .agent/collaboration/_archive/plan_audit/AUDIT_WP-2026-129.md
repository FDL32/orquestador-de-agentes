# AUDIT WP-2026-129 - Review env inheritance fix

## Objetivo de auditoria
Verificar que el entorno del review ya no redirige `HOME`, `USERPROFILE` ni `CODEX_HOME` a `.codex` y que OpenCode puede arrancar con la herencia normal del proceso.

## Criterios funcionales
- `_review_env()` no reescribe las variables de home hacia `.codex`.
- `_get_manager_backend()` ya no devuelve `"codex"` como fallback y usa `"opencode"`.
- OpenCode sigue recibiendo un entorno utilizable para reviews.
- Las rutas de review siguen resolviendose correctamente con el entorno heredado.
- Los tests detectan una regresion si reaparece el redirect.

## Criterios arquitectonicos
- El entorno del review no debe esconder el home real del proceso salvo que exista un motivo especifico y aislado.
- La configuracion de backend no debe quedar acoplada a una home forzada.
- La correccion no debe introducir otra fuente de drift entre backend y review bridge.

## Verificacion esperada
- `ruff check .`
- `pytest`
- `python .agent/agent_controller.py --validate --json --force`

## Archivos esperados
- `bus/review_bridge.py`
- `tests/test_manager_review_bridge.py`
- `tests/unit/test_review_env.py`

## Resultado esperado
- OpenCode puede ejecutar el review sin que `_review_env()` reescriba su home.
- El helper de entorno queda trazable y cubierto por pruebas.
