# AUDIT_WP-2026-142 - Symmetric mark-ready scope gate

## Audit Focus

Verify that `--mark-ready` can no longer accept fabricated scope declarations where the plan lists files that were not actually touched.

## What Must Be True

- Files changed outside the whitelist still block closeout.
- If none of the declared files appear in the diff, the gate blocks.
- If only some declared files appear, the gate warns but does not block.
- `changed_files is None` remains a pass-through for non-git contexts.
- An empty whitelist remains a pass-through.
- Bus emission on `--mark-ready` still works end-to-end after the gate change.

## Failure Modes

1. The new symmetric check blocks valid documentation-only tickets with no meaningful diff overlap.
2. The gate still allows fabricated plans with zero overlap.
3. Partial overlap is treated as a hard error even though the agreed policy is warning-only.
4. The bus emission path regresses because the gate result shape changes unexpectedly.
5. The no-git or empty-whitelist fallbacks stop behaving as before.

## Review Checklist

- Confirm the scope gate compares both directions.
- Confirm the severity split is block vs warning.
- Confirm the tests cover zero-overlap, partial-overlap, out-of-scope, and base cases.
- Confirm `--mark-ready` still produces the expected `STATE_CHANGED` path in the bus tests.
