# AUDIT_WP-2026-132 - Session-close observations for self-improving memory

## Verificacion inicial
La idea encaja con el cierre de sesion existente, pero hoy no hay una skill dedicada que convierta el cierre en observaciones curadas y reutilizables. El objetivo del ticket es formalizar esa pieza sin mezclarla con la consolidacion de memoria.

## Hallazgos
- `skills/project-finalize/SKILL.md` ya tiene un paso opcional de consolidacion de memoria.
- `skills/memory-consolidate/SKILL.md` cubre la promocion y limpieza de `observations.jsonl`, no la generacion de observaciones de cierre.
- `skills/README.md` aun no indexa una skill neutral de cierre de sesion.
- No existe una skill compartida que explique el schema de las observaciones de cierre ni los filtros de promocion.
- El lugar correcto de insercion en `project-finalize` es entre el Paso 9c (`local_audit.py`) y el Paso 9d (`memory_consolidate.py`).
- `validate_all.py` validara el frontmatter YAML de la nueva skill; si el bloque `---` o los campos requeridos faltan, el ticket fallara en validacion.

## Riesgo observado
Si el ticket intenta absorber la consolidacion de memoria, el scope crece y se mezcla una capa de generacion con una capa de promocion. Eso haria mas dificil mantener una memoria estable y auditable.

## Verificacion esperada
- La nueva skill documenta el flujo de cierre de forma compacta.
- La helper script y los tests validan el schema minimo y los filtros, incluyendo categoria invalida, campo requerido ausente, opinion/trace efimera y duplicado deterministico.
- `project-finalize` inserta la skill entre los pasos 9c y 9d.
- `skills/README.md` incluye la nueva skill compartida.
