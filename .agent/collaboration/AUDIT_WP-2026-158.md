## Lo que ya existe
- `AP-12` ya define el problema: un packet basado solo en `git diff` puede ocultar entregables no rastreados.
- `bus/review_bridge.py` ya genera el packet y hoy usa `git diff` + provenance.

- `reviewdog` aporta el vocabulario para `filter_mode` y severidad.

## Blockers
- `bus/review_bridge.py::_prepare_review_packet()` no existe o no se usa como ensamblador central del packet -> Builder debe crear ese helper nuevo y dejar `_build_review_prompt()` como orquestador.
- El packet sigue mostrando solo `git diff` sin seccion `--- Untracked Deliverables ---` o equivalente -> Builder debe anadir la seccion despues del diff y antes del bloque final.
- Untracked visibles en el repo temporal no se filtran y el packet mete ruido de `.agent/collaboration/` o `.agent/runtime/` -> Builder debe excluir artefactos generados y conservar solo deliverables reales.
- `filter_mode` no tiene semantica ni default claros -> Builder debe implementar `diff_context` como default, `added` cuando haya untracked elegibles y `nofilter` solo como respaldo.
- Los tests dependen del repo anfitrion en vez de `tmp_path` -> Builder debe aislar el repo temporal y cubrir un caso con `??`.
