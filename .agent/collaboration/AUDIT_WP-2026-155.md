# Auditoria - WP-2026-155

## Focus
Validar que el runtime del motor deje de asumir `project_root = Path(__file__).parent.parent` en los call-sites [A] y pase a una resolucion unica, inyectable y retrocompatible.

## Checklist de Revision

- [ ] `runtime.project_root.resolve_project_root()` es la unica ruta canonica para el root operativo.
- [ ] `--project-root` se parsea antes de cualquier import que dependa del root.
- [ ] `AGENT_PROJECT_ROOT` se usa en hooks y entry points sin CLI propia.
- [ ] Los call-sites [A] no conservan derivaciones locales de `PROJECT_ROOT` / `AGENT_DIR` para decidir el workspace destino.
- [ ] `agent_system/scripts/project_paths.py` no se usa para redirigir el runtime de WP-A.
- [ ] `agents_config.py` mantiene la derivacion local minima necesaria para bootstrap del `sys.path` y no la elimina accidentalmente.
- [ ] `guard_paths.py` queda fuera del alcance [A] porque su resolucion de `agents.json` es engine-rooted [E] por diseno.
- [ ] Los tests cubren CLI, env y fallback derivado.
- [ ] Los tests que cambian `AGENT_PROJECT_ROOT` llaman `clear_cache()` en setup/teardown.
- [ ] Los scripts de categoria [B] y los artefactos [E] no cambian de comportamiento.

## Evidencia Esperada

- Salida limpia de:
  - `python scripts/run_pytest_safe.py --collect-only tests/test_project_root_resolution.py`
  - `python scripts/run_pytest_safe.py tests/test_project_root_resolution.py -q`
  - `python .agent/agent_controller.py --validate --json --force`

## Criterios de Bloqueo

- Cualquier nuevo call-site [A] que vuelva a derivar `project_root` localmente.
- Cualquier cambio que fuerce a los scripts de instalacion [B] a usar el resolver runtime del motor.
- Cualquier regresion que rompa la precedencia `CLI > env > derivado`.
