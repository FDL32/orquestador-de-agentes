# PLAN_WP-2026-130 - Manager legacy naming cleanup

## Objetivo
Renombrar las referencias legacy que usan `codex` para describir la ruta del Manager, sin tocar la compatibilidad real del backend `codex` configurado en `agents.json`.

## Alcance
- `bus/review_bridge.py`
- `tests/test_manager_review_bridge.py`
- `tests/test_launch_agent_terminals_script.py`
- `templates/startup/manager_legacy.md`
- `.agent/collaboration/work_plan.md`
- `.agent/collaboration/execution_log.md`
- `.agent/collaboration/STATE.md`
- `.agent/collaboration/TURN.md`
- `PROJECT.md`

## Decision Arquitectonica
- `_run_codex_review()` es una ruta legacy del Manager y debe recibir un nombre semantico que no mezcle rol con backend.
- `parse_method = "legacy_codex"` debe pasar a una etiqueta que describa la ruta legacy del Manager.
- Los tests deben reflejar la nomenclatura nueva.
- El backend `codex` real sigue existiendo como opcion de configuracion; este ticket solo limpia el naming del Manager.

## Fases
1. Renombrar la ruta legacy del Manager y su trazabilidad interna.
2. Ajustar fixtures, tests y templates que siguen usando la nomenclatura legacy.
3. Validar que el rename no rompe la compatibilidad del backend `codex` real.

## Criterios de aceptacion
- Ninguna ruta del Manager se presenta como `codex` en nombres internos o trazabilidad.
- Los tests y templates reflejan la semantica correcta.
- La configuracion del backend `codex` real sigue funcionando.
- `ruff`, `pytest` y `python .agent/agent_controller.py --validate --json --force` pasan.
