# Plan de Trabajo - WP-2026-169

## Resumen
Este ticket cierra el loop de session-close desde el controlador principal. El objetivo es exponer `--session-close` en `.agent/agent_controller.py` para que el cierre canonico pase por un unico entrypoint, reuse `scripts/session_closeout.py` y deje las proyecciones listas para el siguiente ciclo sin pasos manuales.

## Objetivo
- Exponer `--session-close` en `agent_controller.py`.
- Delegar la ejecucion al orquestador de cierre existente.
- Sincronizar las proyecciones post-cierre para evitar drift.
- Actualizar la documentacion para que el entrypoint canonico quede visible.

## Reglas de diseno
- No duplicar la pipeline de `scripts/session_closeout.py`.
- No introducir dependencias nuevas.
- No alterar la cascada de `--manager-approve`.
- Mantener `--dry-run` y `--skip-slow`.
- El wrapper debe ser idempotente si el cierre ya se completo.
- Si `STATE.md` ya refleja `COMPLETED` y no se pasa `--force`, el handler debe salir con exit 0 sin relanzar el cierre.

## Fases
### Fase 1
- `agent_controller.py` debe aceptar `--session-close` y delegar a `scripts/session_closeout.py`.
- Los flags `--project-root`, `--dry-run`, `--skip-slow`, `--ticket` y `--tickets` se reusan sin reinterpretacion.
- Tras un cierre real, el handler sincroniza el estado canonico antes de terminar.

### Fase 2
- Cubrir el nuevo entrypoint con tests en `tests/test_agent_controller.py`.
- Actualizar `PROJECT.md`, `README.md`, `QUICKSTART.md` y `CHANGELOG.md`.

## Archivos likely touched
- `.agent/agent_controller.py`
- `tests/test_agent_controller.py`
- `PROJECT.md`
- `README.md`
- `QUICKSTART.md`
- `CHANGELOG.md`

## Criterios de aceptacion
- `--session-close` existe y delega en el orquestador correcto.
- El cierre real deja `STATE.md` alineado con el bus.
- `--dry-run` no muta el repo.
- La documentacion apunta al nuevo entrypoint como ruta canonica.
- La suite de tests cubre el nuevo flujo de cierre.
