# Work Plan - WP-2026-144

- **Estado:** COMPLETED

- **Titulo:** Destination ticket prefix onboarding
- **Asignado a:** Builder

## Objetivo
Make destination projects self-describing for ticket IDs. The installer should accept `--prefix XXX` and persist it into the destination metadata, validation should warn when a host-project lacks `Ticket prefix`, and the onboarding docs should describe the local `XXX-YYYY-NNN` namespace. The motor repo keeps `WP-YYYY-NNN`.

## Decision Arquitectonica
- `scripts/install_agent_system.py --install --prefix XXX` writes the namespace into the destination `PROJECT.md` and into `motor_destination_link.json`.
- `--sync` preserves the destination prefix and keeps the link file auditable.
- `--validate` warns when `active_profile == host-project` and `PROJECT.md` lacks `Ticket prefix:`.
- `prompts/session_bootstrap.md` uses destination-prefixed examples for plan/audit and ticket IDs.
- Cross-cutting docs (`AGENTS.md`, `README.md`, `QUICKSTART.md`, `RELEASE_CHECKLIST.md`, `CLOSURE_MODEL.md`, `PROJECT.md`) stay aligned with the same contract.
- No change to the motor namespace: `WP-YYYY-NNN` remains for this repo.
