# PLAN_WP-2026-157 - ECC capability pack

## Goal

Add a documentation-first research skill, formalize the AP/observations contract, and create a minimal regression harness for the critical orchestration flows.

## Strategy

1. Create `deep-research` as a standalone skill that produces structured context before opening a WP.
2. Tighten the AP contract in `ap-schema.md` and validate `observations.jsonl` with a lightweight script.
3. Add a minimal eval harness for review bridge, guard paths, scope gate, and requeue regressions.

## Scope Notes

- This ticket intentionally avoids ECC 1:1 replication.
- It does not change the storage format of `observations.jsonl`.
- It does not add new Python dependencies.
- It does not rewrite the Manager/Builder workflow.

## Acceptance Shape

- `deep-research` is discoverable, documented and registered in the skills catalog.
- The AP contract is explicit and the validator fails on malformed observations.
- The eval harness is isolated, deterministic and opt-in via `pytest -m eval`.
- The standard safe suite still passes independently of the eval suite.
