# Work Plan - WP-2026-155

## Metadata
- **ID:** WP-2026-155
- **Estado:** IN_PLANNING

- **Titulo:** Runtime project_root contract hardening
- **Asignado a:** Builder

## Objetivo
Eliminar las derivaciones directas de `project_root` basadas en `__file__` en la capa runtime [A] y consolidar la precedencia canonica `CLI --project-root > env AGENT_PROJECT_ROOT > derivado`.

## Contexto
La auditoria WP-2026-121 ya inventario 22 puntos [A] acoplados a `__file__` y definio el contrato deseado. El ticket prepara el runtime para operar sobre un workspace destino sin asumir que el motor vive dentro de ese mismo arbol.

## Decision Arquitectonica
- La funcion central de resolucion debe ser `runtime.project_root.resolve_project_root()`.
- El patron de migracion en runtime es lazy: las referencias a `PROJECT_ROOT` deben resolverse dentro de funciones o `__init__`, no a nivel modulo.
- Los entry points de runtime que aceptan `--project-root` deben exportar `AGENT_PROJECT_ROOT` temprano y preservar fallback compatible.
- Los hooks sin CLI propia deben leer `AGENT_PROJECT_ROOT` y caer al derivado solo si no existe.
- `agents_config.py` conserva una derivacion local minima solo para bootstrap del `sys.path` y para importar `runtime.project_root`; no debe eliminarse ese paso inicial.
- `agent_system/scripts/project_paths.py` se mantiene como helper de instalacion y diagnostico; no se reutiliza para enrutar el runtime [A].
- `guard_paths.py` usa derivacion `__file__` para localizar `agents.json` dentro del motor; es una ruta [E] intencionalmente fuera de alcance.
- La implementacion debe ir en commits atomicos por modulo o por grupo funcional, para mantener revert quirurgico.
