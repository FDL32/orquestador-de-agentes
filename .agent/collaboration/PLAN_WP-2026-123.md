# PLAN_WP-2026-123 - Workspace minimo del destino

## Estrategia

WP-2026-123 publica el contrato de destino minimo sobre la base ya cerrada de
WP-2026-122. La meta es que el instalador produzca un workspace portable con
solo las superficies necesarias del destino, sin copiar el motor operativo.

## Enfoque tecnico

1. Formalizar el archivo de enlace motor-destino con schema explicito.
2. Hacer que `scripts/install_agent_system.py` escriba el `.agent/` minimo.
3. Mantener `MANIFEST.workspace` como allowlist declarativa del destino y
   conservar el enlace generado.
4. Ajustar tests y documentacion para que el contrato quede estable.

## Invariantes

- El motor no se copia al destino.
- `active_profile` del destino queda en `host-project`.
- `MANIFEST.workspace` sigue siendo la referencia de lo que se conserva.
- `MANIFEST.workspace` incluye el enlace generado para que sobreviva a
  `--sync --prune`.
- `install_agent_system.py` sigue siendo el bootstrapper del destino.
