# PLAN_WP-2026-145 - Deterministic STATE projection probe

## Goal

Validate whether the event bus can deterministically reconstruct the current `STATE.md` for the active ticket, without changing the canonical write path yet. The probe should be read-only and should surface drift clearly.

## Strategy

1. Build a standalone probe that reads `events.jsonl` for the active ticket, feeds each `STATE_CHANGED` event through `bus/state_machine.py`, and materializes the final derived state.
2. Compare the derived state against the current `STATE.md`.
3. Add tests for matching state, drift detection, and missing/empty bus cases.
4. Keep `TURN.md` under controller ownership and do not mutate canonical collaboration files.
5. Use the result to decide whether a later WP should promote the probe into a central projector.

## Scope Notes

- This ticket is a research probe, not runtime integration.
- `TURN.md` remains controller-managed.
- The probe should consume `bus/state_machine.py` as the transition authority instead of reimplementing transitions.
- The output is a report and validation signal, not a write-back to canonical markdown.

## Acceptance Shape

- The probe reproduces the canonical `STATE.md` state for sample event streams.
- Drift is reported when the bus-derived state and markdown state disagree.
- The probe is read-only.
- Tests cover happy path, drift, and empty/missing bus cases.
- The probe filters events by `ticket_id` correctly and ignores events from other tickets.
- Events that produce invalid transitions are logged as warnings, not exceptions; the probe continues and reports the anomaly.
