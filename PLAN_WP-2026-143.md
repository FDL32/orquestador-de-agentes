# PLAN_WP-2026-143 - Bus-backed mark-ready idempotency

## Goal

Make `--mark-ready` idempotent by trusting the bus as the source of truth for ticket state. The command should not emit duplicate closeout events when the bus already reflects a ready or terminal-adjacent state.

## Strategy

1. Read the current ticket state from the bus before evaluating the markdown-based path.
2. Treat `READY_FOR_REVIEW` as the existing idempotent "already ready" case.
3. Treat `READY_TO_CLOSE` and `COMPLETED` as clean no-op outcomes.
4. Keep the current markdown fallback only when the bus is unavailable or cannot yield a state.
5. Preserve scope gate, circuit breaker and existing bus emission contracts for the normal implementation path.
6. Extend tests so the command is validated against both the bus-backed no-op path and the legacy fallback path.

## Scope Notes

- The scope gate itself is out of scope for this ticket.
- Circuit breaker behavior stays untouched.
- The Manager approval cascade remains unchanged.
- The change is about avoiding duplicate Builder emissions and review loops.

## Acceptance Shape

- `--mark-ready` exits without duplicate events when the bus already says `READY_FOR_REVIEW`, `READY_TO_CLOSE` or `COMPLETED`.
- The fallback path still works when bus state cannot be read.
- Existing bus emission tests continue to pass.
- New tests verify the no-op behavior explicitly.
