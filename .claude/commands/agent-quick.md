Usa el flujo **quick** para tareas pequeñas.

1. Confirma que el cambio afecta 1-2 archivos, dura menos de 30 minutos y no requiere decisiones de arquitectura.
2. Si no cumple esas condiciones, detente y redirige al usuario a `/agent-plan`.
3. Lee completos los archivos objetivo antes de editarlos.
4. Usa rutas absolutas en tool calls.
5. Implementa el cambio directamente sin plan formal largo.
6. Ejecuta primero el test o validación más cercana al archivo tocado.
7. Si pasa, ejecuta el gate global relevante antes de cerrar el trabajo.
